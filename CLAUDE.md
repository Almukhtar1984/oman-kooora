## Backend (`omkoora-backend--main/`)

- Entry: `src/app.mjs` — Express + Apollo on port `7000` (override with `PORT`). Static `/images` route serves `../uploads/` and falls back to the production CDN.
- Config: `src/Config/DBContact.mjs` (Sequelize → MySQL `tomoh`), `src/Config/logger.mjs` (Winston).
- Models: `src/Models/*.mjs` — Sequelize models with `paranoid: true` (soft delete), timezone `+02:00`.
  - Players, Team, Club, Members, TechnicalApparatus, ClubManagement
  - Request, Transfer, Attachment, AttachmentPerson
  - Message, Comment, Notification, ActionLogs (audit)
  - Permission (14 permission flags as `STRING(25)` columns)
- GraphQL: `src/Graphql/Schemas/*.mjs` + `src/Graphql/Resolvers/*.mjs` (mirrored structure, one file per domain).
- Uploads: `graphql-upload`, 10 MB / 10 files max. Files written to `../uploads/` with UUID names; allowed types include images, PDF, Office, ZIP. Production path: `/www/wwwroot/api/uploads/` (~14 GB, never delete).
- Notifications: only triggered server-side via helpers in `src/Helpers/Notification.mjs` (`CreateNotificationTeam`, `CreateNotificationClub`) which emit through Socket.io (`src/Socket/`). The GraphQL API exposes only read + `markNotificationsAsRead`.
- Audit: `ActionLogs` model exists but has no resolver/schema yet — recording happens inside mutations.

## Frontends — common patterns

All client apps share the same shape:

- `pages/` — Pages Router. Auth via Zustand store (`store/`); on 401 the Apollo `errorLink` retries `getNewToken`.
- `lib/graphql.ts` — Apollo client. URI is `${process.env.NEXT_PUBLIC_API_URL}/graphql` (env var set on the server, not in repo).
- `lib/helpers/image.ts` — image URL helper that prefers `NEXT_PUBLIC_API_URL` then falls back to `NEXT_PUBLIC_UPLOAD_URL`.
- `graphql/queries/<domain>/*.tsx` — `gql` documents.
- `graphql/hooks/<domain>/use<Op>.tsx` — typed hooks generated by GraphQL Codegen (`codegen.ts`, schema cached in `graphql.schema.json`).
- `components/Modal/` holds the bulk of the UI (60+ modals in club/team).
- `components/Tables/`, `Drawer/`, `Layout/`, `Card/`, `Input/`, `Select/`, `Tabs/`, `PDF/` (PDFs use `@react-pdf/renderer`).

### Where things live (club/team apps)

- Players: `pages/players.tsx`, `components/Tables/PlayersTable.tsx`, modals `AddPlayerModal`, `UpdatePlayerModal`, `ShowAttachments`, `AddAttachmentPlayerModal`, `DeleteAttachmentPlayerModal`, `VerifyIdentityModal`, `ChangeClassificationModal`, `ConvertPlayerToTechnicalModal`.
- Teams: `pages/team.tsx`, `pages/team/[id].tsx`, `components/Modal/AddTeamModal.tsx`.
- Members / Tech-staff: `pages/members.tsx`, `pages/technicalApparatus.tsx`, with corresponding `Add*` / `Update*` / `ChangeStatus*` modals.
- Requests / Transfers: `pages/requests.tsx`, `players_transfer.tsx`, `players_loan.tsx`. Reject reason flows through `UpdateRequestStatusModal` (`note` field).
- Messages: `pages/messages.tsx` (inbox + outbox tabs), `Tables/InboxTable.tsx`, `Tables/OutboxTable.tsx`, `Modal/AddMessage.tsx`, `Modal/AddCommentModal.tsx`, `Drawer/CommentDrawer.tsx`.
- Permissions UI: `pages/powers.tsx` + `useAllPermission` / `useAddPermission` / `useUpdatePermission` hooks.
- Print: `components/Modal/PrintModal.tsx` (club only).

## Production deployment

- VPS: Contabo, `62.171.179.219`, Ubuntu 22.04, aaPanel + Nginx (not Apache).
- Repo mirror on server: `/www/wwwroot/_repo_omkoora/` (deploy does `git pull` here).
- Deploy script: `/root/scripts/deploy.sh [api|club|team|super-admin|player|all]`.
- Backend: PM2 on port 7000.
- Frontends: 3001 (club), 3002 (team), 3003 (super-admin), 3004 (player); landing is static under `/www/wwwroot/landing/out/`.
- Node versions used on the server: `v18.17.0` for `pm2`, `v18.20.0` for `npm install` / `npm run build`.
